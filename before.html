BEFORE Trigger

When defining a trigger, you can specify the trigger timing. That is, you can specify whether the trigger action is to be executed before or after the triggering statement. BEFORE and AFTER apply to both statement and row triggers.

BEFORE triggers are commonly used to check the validity of the data before the actions performed. For instance, you can use BEFORE trigger to prevent deletion of row if deletion should not be allowed in the given case.

EXAMPLE

CREATE A TRIGGER FOR THE EMP TABLE, WHICH MAKES THE ENTRY IN NAME COLUMN IN UPPERCASE 
CREATE OR REPLACE TRIGGER UPPER
BEFORE INSERT ON EMP
FOR EACH ROW
 BEGIN
:new.NAME := UPPER(:new.NAME);
 END;
